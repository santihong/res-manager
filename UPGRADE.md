# 🎉 v1.1.0 升级说明

## 新功能：时间戳目录管理

每次下载的图片现在会自动保存到独立的时间戳目录中！

### 📁 新的目录结构

**之前（v1.0.0）：**
```
Downloads/images/
├── image1.jpg
├── image2.png
├── photo1.jpg
└── photo2.jpg
```
所有图片混在一起，难以管理 😵

**现在（v1.1.0）：**
```
Downloads/images/
├── 20240213_143025/
│   ├── image1.jpg
│   └── image2.png
└── 20240213_154510/
    ├── photo1.jpg
    └── photo2.jpg
```
每次下载都有独立目录，清晰明了！✨

### 🎯 时间戳格式

`YYYYMMDD_HHMMSS`

例如：`20240213_143025` 表示 2024年2月13日 14:30:25

### ✅ 优势

1. **独立存放** - 不同批次的图片不会混淆
2. **时间追溯** - 一眼就能看出下载时间
3. **便于管理** - 可以按时间整理和清理图片
4. **自动化** - 无需手动创建目录

### 🔄 如何升级

#### 方法1：重新加载（推荐）

1. 打开 `chrome://extensions/`
2. 找到「图片资源下载器」
3. 点击「重新加载」按钮（🔄图标）
4. 完成！

#### 方法2：重新安装

1. 在 `chrome://extensions/` 删除旧版本
2. 重新加载插件目录
3. 完成！

### 📝 使用方法

**完全不变！** 使用方式和之前一模一样：

1. 点击插件图标
2. 选择图片格式
3. 点击下载按钮
4. 图片自动保存到时间戳目录 ✨

### 💡 提示

- 成功提示会显示具体的保存目录
- 例如：`成功下载 15 张图片到 images/20240213_143025/ 目录！`

### ⚠️ 注意事项

- 同一秒内的多次下载会使用同一个目录
- 旧版本下载的图片不受影响
- 如果需要修改目录格式，可以编辑 `popup.js` 中的 `generateTimestampFolder()` 函数

### 🐛 遇到问题？

如果升级后遇到问题：

1. **清除缓存**：在 `chrome://extensions/` 点击「重新加载」
2. **查看日志**：右键插件图标 → 「检查弹出内容」→ Console
3. **重新安装**：删除后重新加载插件
